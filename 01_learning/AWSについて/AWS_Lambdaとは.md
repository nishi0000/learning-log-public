

# テーマ

AWS Lambda（ラムダ）とは何か／APIリクエストはどこで受け取られているのか

## 目的

AWSでよく出てくる「Lambda」という言葉が、  
サーバーなのか、APIなのか分からず混乱していたため、  
**実際に何が起きているのか**を整理する。

## 本筋

- Lambdaは何をするものか
    
- URL（APIリクエスト）は誰が受け取っているのか
    
- サーバーとの違いは何か


## 結論

Lambdaはサーバーではない。  
URLを受け取るのは **API Gateway** で、  
Lambdaは **そのリクエストをきっかけに、登録されたプログラムを一瞬だけ実行する仕組み**。

自分がサーバーを持っているわけではなく、  
AWSが用意しているサービスを使って処理を動かしている。

## 図解

```text
利用者（ブラウザ / アプリ / intra-mart）
   ↓ URLアクセス
API Gateway（AWSが常に待機している）
   ↓ リクエストを渡す
Lambda（その瞬間だけ起動して処理を実行）
   ↓ 実行結果を返す
API Gateway
   ↓
利用者
```

## 詳細

- Lambdaは常に動いているわけではない
    
- 何かのきっかけ（例：API Gateway経由のリクエスト）が来たときだけ起動する
    
- 処理が終わると自動で終了する
    
- URLを直接受け取ることはできない
    
- URLを持ち、リクエストを受け取る役割は API Gateway が担っている
    
- サーバーの起動・停止・常駐管理は自分では行っていない

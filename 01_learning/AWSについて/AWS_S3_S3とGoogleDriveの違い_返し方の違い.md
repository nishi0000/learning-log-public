
# テーマ

**AWS S3（静的Webホスティング／配信の仕組み）**

---
## 目的

- Vue を build してできた `dist` を S3 に置いたが、  
    **なぜ S3 では Web ページとして表示できて、  
    Google Drive ではできないのか**を理解したかった
- 「置き場所」や「URLがあるか」ではなく、  
    **サービスの性質の違い**が理由だと整理したかった

---
## 本筋

- S3 は「ただの保管庫」なのか？
- なぜ `index.html` にアクセスできるのか？
- なぜ Google Drive では同じことができないのか？
- この違いは、どこから来ているのか？

---

## 結論

- **S3 は、アクセスされたらファイルの中身をそのまま返すサービス**    
- **Google Drive は、アクセスされたファイルを解釈して、  
    Google の UI（プレビュー／編集／DL 画面）を返すサービス**    
- そのため
    - S3 に置いた HTML → ブラウザがそのままページとして表示できる
    - Google Drive に置いた HTML → ページとしては表示されない
- これは  
    **「サービスの性質（どう返すか）」の違い**によるもの

---
## 図解

```text
【S3 の場合】

ブラウザ
  ↓（HTMLください）
S3
  ↓（そのまま返す）
HTMLの中身
  ↓
ブラウザがWebページとして表示


【Google Drive の場合】

ブラウザ
  ↓（HTMLにアクセス）
Google Drive
  ↓（これはファイルだな、と解釈）
GoogleのUI（プレビュー／編集／DL画面）
  ↓
Webページとしては表示されない
```

---
## 詳細

- `npm run build` で生成される `dist` は  
    **ブラウザが直接読む前提のファイル（HTML / JS / CSS）**
    
- S3 は  
    **リクエストされたファイルを加工・解釈せずに、そのまま返す**
    
- Google Drive は  
    **「人がファイルを扱う」ためのサービスなので、  
    ファイルにアクセスすると Google 側の画面を返す**
    
- 同じ HTML ファイルでも  
    **「どこに置いたか」ではなく  
    「どんなサービスが返しているか」で結果が変わる**

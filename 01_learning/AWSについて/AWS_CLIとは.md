
# テーマ

AWS CLIとは何か、画面操作（AWSコンソール）と何が違うのか

## 目的

AWSを使うときに出てくる「CLI」という言葉の正体をはっきりさせて、  
なぜインストールしたのか・いつ使うものなのかを混乱なく理解するため。

## 本筋

- CLIとは何の略か
    
- AWSの画面操作とCLIで、実際に何が違うのか
    
- なぜ自分はCLIをインストールしたのか
    

## 結論

CLIは「文字を打って操作する方法」のこと。  
AWS CLIは、AWSを画面ではなく自分のPCから文字の命令で操作するための道具。  
AWSの中で起きている処理は、画面操作でもCLIでもほぼ同じで、  
**入口（操作のしかた）が違うだけ**。  
Lambdaのテンプレート取得やデプロイのように、  
コードを扱う作業はCLIのほうが早くて楽なのでインストールした。

## 図解

```text
AWS
 ↑        ↑
画面操作   CLI
（ブラウザ） （自分のPC）

どちらも同じAWSに命令している
```

## 詳細

- CLIは「Command Line Interface」の略で、文字入力で操作する方法
    
- AWS CLIはAWS専用のCLIツール
    
- 画面操作は人間に分かりやすいが、同じ作業を何度もやると面倒
    
- CLIはコマンドをコピペ・再実行できるので、再現性が高い
    
- Lambdaの雛形取得やデプロイのような作業はCLIのほうが効率がいい
    
- AWSはCLIを使わなくても、画面操作だけでも利用できる

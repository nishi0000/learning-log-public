
# テーマ

同期処理と非同期処理の違いは何か？

## 目的

- 「同期処理／非同期処理」を言葉で説明できるようになるため
    
- await や非同期ループで感じていたモヤっとを整理するため
    

## 本筋

- 同期処理と非同期処理の**本質的な違い**をつかむ
    
- 「待つ／待たない」という言葉の誤解を解消する
    

## 結論

同期処理と非同期処理の違いは、  
**処理が終わるまで「席をふさいで待つか」「席をあけて待つか」の違い**。

- 同期処理  
    → 処理が終わるまで、その場（処理枠）を占有したまま待つ
    
- 非同期処理  
    → 処理が終わるのを待つが、その間はいったん席をあけて、あとで戻ってくる
    

`await` を使って「待っている」場合でも、  
**席をあけて待っているなら非同期処理**。

## 図解

```text
同期処理：
[ 処理A が席を占有 ] ──待ち── [ 処理B ]
（この間、他の処理は進めない）

非同期処理：
[ 処理A 開始 → 登録 ] → 席をあける → 他の処理
                         ↑
                   終わったら戻ってくる
```

## 詳細

- プログラムはどちらの場合も上から順に実行される
    
- 違いは「待っている間、その処理枠を占有しているかどうか」
    
- 非同期処理は
    
    - 結果を待たないわけではない
        
    - **待ち方が違う（ブロックしない）**
        
- `await` は
    
    - 非同期処理を同期に変えるものではない
        
    - 「この関数はいったん席を外して待つ」と書いているだけ

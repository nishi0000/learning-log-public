    - 覚えるもの：

-  トピック：CISC / RISC / VLIW / MIMD のレイヤー差
    
    - 何を扱った：
        
        - 「レイヤー（指している対象）が違うのでは？」の整理
            
        - CISCの誤解（“作業台に振り分け”と混同）を修正
            
    - 確定した結論：
        
        - CISC/RISC：CPUが理解する命令セット（ISA）の設計思想（CPU設計の話）
            
        - VLIW：命令形式（CPU側の設計）＋「束ね方はコンパイラが担う」（CPUとコンパイラがセット）
            
        - MIMD：命令形式ではなく「並列計算の分類（複数命令×複数データ）」で、別枠
            
    - 覚えるもの：
        
        - 4語の分類ルール
            
            - CISC/RISC＝命令セット設計（ISA）
                
            - VLIW＝命令形式＋コンパイラ協調
                
            - MIMD＝並列計算の分類（システム形態）
                
-  トピック：プロセッサ／CPU／クロック／高速化の方向性
    
    - 何を扱った：
        
        - プロセッサ＝CPUか？
            
        - クロックとは？
            
        - 高速化の大分類（テンポ↑、1クロックの仕事↑、待ち↓、コア↑）
            
    - 確定した結論：
        
        - プロセッサ ≒ CPU（命令を実行する装置）
            
        - CPUのイメージ：コンピュータの「脳みそ」（命令を取りに行く・解釈する・計算する・メモリとやり取り）
            
        - クロック：CPUの動作の拍（テンポ）。周波数は1秒あたりの拍数
            
        - 高速化の観点（整理）：
            
            - クロックを上げる（テンポ↑）
                
            - 1クロックでできる仕事を増やす（同時実行＝命令レベル並列など）
                
            - 待ち時間を減らす（キャッシュなど）
                
            - コアを増やす（並列に処理）
                
    - 覚えるもの：
        
        - クロック＝テンポ
            
        - 高速化4分類（テンポ↑／同時実行↑／待ち↓／コア↑）
            
-  トピック：「命令はこまぎまのまま同時にこなす」の意味
    
    - 何を扱った：
        
        - 「キャパがあるのに1つずつ渡すのを直す？」の疑問
            
    - 確定した結論：
        
        - 1コアの中にも複数の“作業台”（計算、ロード等）があり、独立な処理は同時に進められる
            
        - 命令の粒度（CISC/RISC）とは別に、「同時実行できるものを並べる」観点がある
            
    - 覚えるもの：
        
        - 同時実行（ILP）の直感：独立な作業は並べられる
            
-  トピック：CPUとコンパイラは別物だが“約束ごと”で接続される
    
    - 何を扱った：
        
        - 「CPUとコンパイラは場所が違うのにセット？」の疑問
            
    - 確定した結論：
        
        - 共通言語：命令セット（ISA）／命令フォーマット
            
        - CPUが受け取れる命令形式に合わせて、コンパイラが命令を生成する（鍵穴と鍵の関係）
            
    - 覚えるもの：
        
        - 「CPUとコンパイラはISAでつながる」という見方

### 2. 確定した定義・式（抜粋）

- 用語：OSS
    
    - 定義：
        
        - Open Source Software
            
        - ソースコードが公開され、ライセンス条件の範囲で利用・改変・再配布が可能なソフトウェア
            
    - 式：なし
        
    - 関連する用語（このスレッド内の範囲）：
        
        - ライセンス（条件を守る必要がある、という話題に触れた）
            
- 用語：Scala
    
    - 定義：
        
        - プログラミング言語（読み：スカラ）
            
    - 式：なし
        
    - 関連する用語：
        
        - JVM（ScalaはJVM上で動く）
            
        - 言語カテゴリ（オブジェクト指向＋関数型の性質）
            
- 用語：JVM
    
    - 定義：
        
        - Java Virtual Machine（Java仮想マシン）
            
        - Java/Scala等のバイトコードを実行する環境
            
    - 式：なし
        
    - 関連する用語：
        
        - .NET / CLR（対応づけの話）
            
- 用語：VLIW
    
    - 定義：
        
        - Very Long Instruction Word
            
        - コンパイル時に同時実行可能な操作をまとめて「1つの長い命令」としてCPUへ渡す方式
            
    - 式：なし
        
    - 関連する用語：
        
        - 命令レベル並列（同時実行）
            
        - CPUとコンパイラの協調（命令フォーマット共有）
            
- 用語：CISC
    
    - 定義：
        
        - Complex Instruction Set Computer（複雑な命令セットの設計思想）
            
        - CPU命令が多機能・複雑寄り
            
    - 式：なし
        
    - 関連する用語：
        
        - RISC（対比）
            
- 用語：RISC
    
    - 定義：
        
        - Reduced Instruction Set Computer（単純化された命令セットの設計思想）
            
        - CPU命令が単純寄り
            
    - 式：なし
        
    - 関連する用語：
        
        - CISC（対比）
            
- 用語：MIMD
    
    - 定義：
        
        - Multiple Instruction, Multiple Data
            
        - 並列計算の分類（複数命令×複数データ）
            
    - 式：なし
        
    - 関連する用語：
        
        - 「命令形式」ではなく「並列形態」というレイヤー差
            
- 用語：クロック
    
    - 定義：
        
        - CPUの動作の拍（テンポ）
            
        - 周波数＝1秒あたりの拍数
            
    - 式：
        
        - 例示：3GHz = 1秒あたり約30億回の拍（説明として扱った）
            
    - 関連する用語：
        
        - 高速化技法（テンポ↑の観点）

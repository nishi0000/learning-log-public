
# 実装ログ（事実ログ）：GitHub公開用の整理（Vue + Lambda + Obsidian）

## 1. 対象

- Vue（Vite）フロントエンド + AWS Lambda（C#/.NET）バックエンドのプロジェクトをGitHubで管理するための整理
    
- Obsidian（技術ログ）をPublic用に分離してGit管理するための整理
    

---

## 2. 実施内容サマリ（確定した状態）

- リポジトリ構成を `frontend/` と `backend/` に分離（モノレポ構成）
    
- `.gitignore` を整備し、生成物・依存物・ローカル設定をGit追跡対象から除外
    
- `git init` → `git status` で対象確認 → 初回コミット準備まで実施
    
- GitHubでリポジトリ削除→復元（restore）を実施
    
- ObsidianのPublic用Vaultを新規作成し、Gitプラグイン（Obsidian Git）を導入
    

---

## 3. ファイル/ディレクトリ構成の整理

### 3.1 採用したリポジトリ構成（最終）

```
MYAPP/
  frontend/
  backend/
    TodoApi/
  infra/
  myapp.sln
  .gitignore
```

### 3.2 フォルダ移動（実施）

- Vue側のフォルダを `frontend/` 配下に配置
    
- Lambda側の `TodoApi` を `backend/TodoApi` に移動
    
- 途中、フォルダのリネーム/移動でエラーが発生し（EPERM）、移動が完了するまで再試行した
    

---

## 4. 発生したエラーと結果

### 4.1 フォルダ移動時のエラー（EPERM）

- `TodoApi` を `backend\TodoApi` に移動（rename）しようとした際に `EPERM` が発生
    
- エラー表示は `rename '...TodoApi' -> '...backend\TodoApi'` 形式
    
- 対応後、`backend/TodoApi` への移動が完了

## 6. Git操作（実施）

### 6.1 `git init` 実施

- VSCodeの操作で `git init` を実施
    
- ソース管理画面が有効化され、コミット待ち状態になった
    

### 6.2 `git status` の確認

- `git status` 実行結果で `Untracked files` が以下のみであることを確認

    - `backend/`
        
    - `frontend/`
        
    - `myapp.sln`
        
- `node_modules` や `dist` が `Untracked files` に出ていないことを確認
    

### 6.3 初回コミット準備（手順として実施/提示）

- `git commit -m "Initial commit"`

## 7. GitHub側操作（実施）

### 7.1 Publicリポジトリ作成方針

- Publicで運用する前提で、新規リポジトリ作成 → ローカルからpushする流れを採用
    
- GitHub側ではREADME等の初期生成はOFFで作成する方針
    

### 7.2 リポジトリ削除と復元

- GitHubの画面上でリポジトリを削除
    
- 削除直後、復元一覧に表示されない状態を確認
    
- その後、リポジトリの復元（restore）が完了
    

---

## 8. Obsidian（技術ログ）をPublic用に分離

### 8.1 Public用Vault作成

- Obsidianで新しい保管庫（Vault）を作成
    

### 8.2 プラグイン導入

- Obsidian Git プラグインをインストール
    

### 8.3 Public Vault側の `.gitignore`

- Public Vault内で除外したいディレクトリを `.gitignore` に記載
    

例：

```gitignore
/00_template/
/98_doc/
/99_meta/
/.obsidian/
```

### 8.4 `.gitignore` のコミット

- `.gitignore` をリポジトリに含めた（アップロード/コミット対象にした）
